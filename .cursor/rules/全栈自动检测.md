# 盛安密封 · 新增代码强制自动检测规则（Cursor 全局生效）

> 适用项目：盛安密封全栈网站（Flask 后端 + React/Vite 前端，部署于小红 VPS `/var/www/shengan-seal`）
> 本文件放入 `.cursor/rules/` 后，Cursor 每次生成 / 修改代码都会自动执行以下自检，**不允许在交付前跳过检测**。

---

## 一、本项目的真实可用工具（已安装、已验证）

- **Semgrep**（静态扫描）：`semgrep scan --config ~/.workbuddy/skills/ai-site-audit/semgrep-rules.yaml backend src index.html`
- **ai-site-audit**（自研全量审计）：`python3 ~/.workbuddy/skills/ai-site-audit/scan.py .`
- **底层门禁 = 上述两者**，用于门禁代码是否违背三条强制约束。

## 二、明确剔除的无效工具（本项目不适用，禁止调用）

- ❌ **Spectral / OpenAPI 校验**：后端**没有 `openapi.json` / Swagger 文档**，无被校验源，调用即空转。
- ❌ **`specify validate` / `specify analyze`**：本机 `specify` 二进制**无 `validate` 子命令**（脚手架也只装了 `speckit` 工作流）。Spec-Kit 仅作规范底座，不具代码扫描能力。

## 三、每次改动必须执行的三项校验

1. **危险 ORM / 数据库配置**
   - 禁止 `sync({force:true})`、`db.drop_all()`、`DROP TABLE`、启动即 `CREATE TABLE IF NOT EXISTS` 之外的删表逻辑。
   - 数据库结构变更只允许迁移脚本，禁止重启清空数据表。
2. **前端业务硬编码**
   - 禁止页面写死：公司名称、电话/微信、产品文案、静态图片 CDN 地址、拼多多等外链。
   - 一律由后端 `/api/settings` 动态获取；允许存在的仅限 `settings.x || '默认值'` 这类**安全兜底**。
3. **前后端字段对齐**
   - 数据库模型字段、接口返回 JSON 字段、前端渲染取值变量命名一一对应。
   - 前端组件**内部私有属性**（如 `data_key` / `for_each` / `is_route` / `text_content`）属框架驱动渲染，不计入错位，单独备注即可。

## 四、输出与拦截要求

- 输出本次改动的风险清单：**文件路径:行号 | 风险等级 | 业务影响 | 修复代码片段**。
- 存在 **Critical / High** 级问题时，**必须同步给出修复代码**，否则不完成代码交付。
- 检测结论须用**独立手段复核**（例如 grep 全源码 + 构建产物 grep），避免检测工具自身漏扫（曾因规则漏 `.html`/`.astro` 导致假阴性）。

## 五、动态业务验证（写完代码后的可选闭环）

仅靠静态扫描查不出运行时问题，下列两项用 Playwright 自动跑（见 `tests/e2e/admin/dynamic-audit.spec.ts`）：
- 后台改配置 → 刷新前台验证同步；
- 新增测试产品 → 重启后端 → 验证数据保留。
