# 盛安密封三大问题修复与 Spec-Kit 安装

**日期：** 2026-07-20
**关联工作日记：** 2026-07-16_盛安密封阶段性复盘与下一步

---

## 一、目标

修复 2026-07-20 体检发现的三个问题：
1. 前端大量硬编码（电话、拼多多链接、公司名写死）；
2. 后台更新与前台错位 + 代码内双种子冲突（app.py 2 款产品 vs init_db.py 8 款产品）；
3. 每次启动服务可能建新 SQL（init_db.py 种子无幂等守卫，手跑即清空后台数据）。

并完成用户要求的 Spec-Kit（specify-cli）安装，用于后续 spec-driven 开发。

## 二、修改范围

### 后端：消除双种子 + 种子幂等化
- `backend/app.py`
  - 删除重复的 `seed_data()` 函数（原 2 款产品种子，与 init_db.py 冲突的根源）。
  - `__main__`（开发入口）改为：`init_db()` 后 `import init_db as _seeder` 并调用 `seed_products / seed_sub_products / seed_settings / seed_qr_codes / seed_news`（统一走 init_db 的幂等种子）。
  - `else`（gunicorn 模块导入）分支只保留 `init_db()`（建表+迁移+清理），**不再灌数据**——避免多 worker 并发启动时竞态重复插入。初始数据由 `deploy.sh` 在 DB 空时调用 `init_db.py` 完成。
- `backend/init_db.py`
  - 5 个 seed 函数（products / sub_products / settings / news / qr_codes）由 `DELETE FROM` + 无条件 INSERT 改为「**仅空表时插入**」：`if SELECT COUNT(*) > 0: db.close(); return`。
  - 种子业务数据对齐真实值：`hotline` 由占位 `400-888-SEAL` 改为 `13507402179`；`company_name` 改为 `长沙盛安密封科技有限公司`（与前端默认值一致）。

### 前端：硬编码改为读取后台 settings
- 新增 `src/lib/settings.ts`：模块级缓存的 `getSettings()` + `useSettings()` 钩子（全站仅打一次 `/api/settings`）。
- 6 个组件改为读后台设置（带 fallback）：
  - `TopBanner.tsx`：电话
  - `FloatSidebar.tsx`：tel 电话 + 两处拼多多链接
  - `Header.tsx`：两处拼多多链接
  - `BannerCarousel.tsx`：`href2` 为 http 时替换为后台 `pdd_link`
  - `OnlineChannel.tsx` / `EngineeringCases.tsx`：拼多多链接
- `ContactFooter.tsx`：「微信客服」电话也改为读 `settings.hotline`。
- 剩余字面量（`13507402179`、`mobile.yangkeduo.com`）均仅作 fallback 兜底，后台设置优先。
