## 二、三种售卖方案（按客户类型选择）

### 方案 A｜最简标准（首推，卖给中小客户）

**适用：** 展示官网、AI 建站、小系统

**核心规则：**
- 所有静态资源统一使用**根相对路径 `/`**
- ❌ 禁止写死：`http://123.45.67.8/img/a.png`
- ✅ 统一写成：`/img/a.png`

**好处：**
1. 客户部署到任意 IP、任意域名、http/https，资源自动适配
2. 客户不需要修改任何图片路径，只需要解析域名
3. 没有跨域、混合内容坑，售后极少
4. 不需要复杂配置，宝塔、Nginx、Apache 全都兼容

**配套强制规则（交付必加，减少售后）：**

在 Nginx 配置里增加：禁止直接用 IP 访问网站

```nginx
# IP 访问强制跳转域名
if ($host != $server_name) {
    return 301 https://$server_name$request_uri;
}
```

**原因：**
- 防止别人直接用 IP 访问客户网站
- 从根源避免"用户习惯用 IP 打开，又出现各种资源异常"

### 方案 B｜工程化标准（卖给技术型客户）

**适用：** SaaS 类、Vue/React 前端网站

**核心规则：**
- 用环境变量管理域名，不要硬编码
- 开发时：环境变量留空或用相对路径
- 客户部署时：只需在 `.env` 配置文件填写他的域名

```env
# .env 客户只改这一行就行
VITE_WEB_DOMAIN=
# 为空就自动使用根路径 /assets/xxx
```

**缺点：** 稍微复杂一点，非技术客户看不懂 .env 配置

### 方案 C｜企业级交付标准（大客户高价项目）

**适用：** 几万、十几万定制网站项目

**核心规则：**
1. 网站业务资源全部脱离主站，走独立 CDN
2. 交付时给客户两套东西：
   - 网站程序包（资源全部写 `/` 路径）
   - 《部署文档》：域名解析、SSL 证书、静态缓存、禁止 IP 访问、CDN 对接步骤

**小源码售卖没必要上 CDN，会增加客户成本。**

### 🎯 最终标准答案（直接照这个做）

**主方案：** 全部资源使用根路径 `/`，Nginx 配置禁止 IP 直接访问，附带一份简易部署文档

1. 所有图片、CSS、JS → `/static/xxx`
2. 不要写任何 IP、不要写死完整域名地址
3. 交付文档写明：部署后配置 SSL 证书，不要通过 IP 访问网站
4. 可选：给客户提供一套一键部署脚本（宝塔导入配置）

### ❌ 绝对不要做（卖网站大忌）

1. ❌ 写死你自己服务器 IP、测试域名在源码里（客户一换服务器直接全部裂图，售后爆炸）
2. ❌ 靠让客户去改 Nginx 跨域头去"修复图片"，属于临时补丁，交付极不专业
3. ❌ 用 `//` 协议自适应路径，部分老旧服务器会有兼容 bug

## 六、部署避坑指南

### 6.1 宝塔面板 Nginx 配置路径

❌ **错误路径：** `/etc/nginx/sites-available/`
✅ **正确路径：** `/www/server/panel/vhost/nginx/`

**示例：**
```bash
# 盛安密封官网配置
/www/server/panel/vhost/nginx/shengan-seal.conf

# 主站反向代理规则
/www/server/panel/vhost/nginx/proxy/maichewei.com/shengan.conf
```

### 6.2 Nginx location 顺序

❌ **错误：** 精确匹配放在通配匹配之后
✅ **正确：** 精确匹配必须放在最前面

**示例：**
```nginx
# ✅ 正确顺序
location = /shengan/sitemap.xml { ... }  # 精确匹配（最前）
location = /shengan/robots.txt { ... }   # 精确匹配
location /shengan/api/ { ... }           # 前缀匹配
location /shengan/ { ... }               # 通配匹配（最后）
```

### 6.3 Nginx images location 必须用 ^~ 前缀

❌ **错误：** `location /images/ { ... }`
✅ **正确：** `location ^~ /images/ { ... }`

**原因：** `^~` 确保前缀匹配优先级高于后续正则匹配（如 `location ~* \.(jpg|png|webp)$`），否则请求会被错误路由。

### 6.4 上传路径需匹配 nginx root 路径

❌ **错误：** 上传到 `/dist/images/products/`
✅ **正确：** 上传到 `/var/www/shengan-seal/images/products/`

**原因：** nginx 配置的 root 为 `/var/www/shengan-seal`，因此图片 URL `/images/products/xxx` 实际对应文件系统路径 `/var/www/shengan-seal/images/products/xxx`。

### 6.5 scp 不支持远程通配符

❌ **错误：** `scp xiaohong:/backup/* ./local/`
✅ **正确：** `scp 'xiaohong:/backup/a.conf' 'xiaohong:/backup/b.conf' ./local/`

**原因：** shell 无法展开远程 glob，必须显式列出每个文件路径。

## 七、开发避坑指南

### 7.1 Flask JSON 字段需显式 json.loads

❌ **错误：** 直接返回数据库中的 JSON 字符串
✅ **正确：** 显式解析 JSON 字符串

**示例：**
```python
# ❌ 错误
products = db.execute("SELECT * FROM products").fetchall()
return jsonify(products)  # gallery_images 是字符串

# ✅ 正确
for item in products:
    item['gallery_images'] = json.loads(item['gallery_images']) if item.get('gallery_images') else []
    item['detail_images'] = json.loads(item['detail_images']) if item.get('detail_images') else []
return jsonify(products)
```

### 7.2 sqlite3.Row 不支持 .get()

❌ **错误：** `row.get('field')`
✅ **正确：** `dict(row).get('field')`

**示例：**
```python
# ❌ 错误
row = db.execute("SELECT * FROM products WHERE id=?", (id,)).fetchone()
name = row.get('name')  # AttributeError

# ✅ 正确
row = db.execute("SELECT * FROM products WHERE id=?", (id,)).fetchone()
rd = dict(row)
name = rd.get('name')
```

### 7.3 Flask generate_token 需兼容 SQLite NOT NULL 约束

❌ **错误：** 只插入 token，不插入 user
✅ **正确：** 插入 token + user，添加 try/except 兼容旧表结构

**示例：**
```python
def generate_token(user='admin'):
    token = secrets.token_hex(32)
    try:
        db.execute("INSERT INTO admin_tokens (token, user) VALUES (?, ?)", (token, user))
    except sqlite3.OperationalError:
        # 兼容旧表结构（无 user 字段）
        db.execute("INSERT INTO admin_tokens (token) VALUES (?)", (token,))
    db.commit()
    return token
```

### 7.4 产品图片上传格式与大小限制

**支持格式：** JPG、JPEG、PNG、GIF、WebP
**大小限制：** 单张不超过 20MB

**前端校验：**

### 10.2 核心代码示例

```python
from flask import Flask, render_template
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///shengan.db'
db = SQLAlchemy(app)

class Product(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100))
    description = db.Column(db.Text)
    image_url = db.Column(db.String(200))

@app.route('/')
def index():
    products = Product.query.all()
    return render_template('index.html', products=products)

@app.route('/products/<int:id>')
def product_detail(id):
    product = Product.query.get_or_404(id)
    return render_template('product_detail.html', product=product)

if __name__ == '__main__':
    app.run()
```

1. **SEO 优先：** 服务端渲染 > 前后端分离

- SEO 最佳（服务端渲染）
- 开发最简单（只需 Python）
- 交付最简单（Python + nginx）
- 维护成本最低
