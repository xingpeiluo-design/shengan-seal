## 一、项目定位与交付标准

### 1.1 项目定位

**核心特征：**
- 零硬编码（无 IP、域名、协议硬编码）
- 客户拿到源码后 3 步上线（解析域名 → 执行 deploy.sh → 申请 SSL）

### 1.2 交付标准

**必须提供：**
1. ✅ 完整源码（含前端 + 后端 + 图片资源）
2. ✅ 一键部署脚本（deploy.sh）
3. ✅ 交付部署手册（DEPLOY.md，含极简版 + 详细版）
4. ✅ SEO 优势说明（面向甲方展示）
5. ✅ 工作日记（完整开发记录）

**交付承诺：**
- 无 IP 硬编码
- 无域名绑定
- 无环境残留
- 换服务器/换域名直接可用

## 二、推荐技术栈（2026 版）

### 2.1 首选方案：Flask + Jinja2 + Tailwind CSS

**为什么推荐这个方案？**

| 维度 | 得分 | 说明 |
|------|------|------|
| **SEO 友好度** | ⭐⭐⭐⭐⭐ | 服务端渲染，爬虫直接抓 HTML，百度收录最佳 |
| **开发难度** | ⭐⭐⭐⭐⭐ | 只需 Python，Jinja2 模板直接输出 HTML |
| **交付难度** | ⭐⭐⭐⭐⭐ | 一个 Python 项目 + nginx，部署极简 |
| **维护成本** | ⭐⭐⭐⭐⭐ | 客户只需 Python + nginx，无需 Node.js |
| **前端定制** | ⭐⭐⭐⭐ | Tailwind CSS 快速开发，无需学 React/Vue |

**技术栈详解：**

```
后端：Flask + SQLite/PostgreSQL
模板：Jinja2（Flask 内置）
样式：Tailwind CSS（CDN 或本地编译）
管理：Flask-Admin（内置后台管理）
部署：Gunicorn + Nginx + Systemd
```

### 2.2 对比盛安密封当前方案（React SPA）

| 维度 | React SPA（当前） | Flask + Jinja2（推荐） |
|------|------------------|---------------------|
| **SEO** | 75/100（需优化） | **90+/100**（天然友好） |
| **首屏加载** | 2-3 秒 | **< 1 秒** |
| **开发难度** | 需 React + Flask | **只需 Python** |
| **部署复杂度** | npm + Python + nginx | **Python + nginx** |
| **后台管理** | 自研 React Admin | **Flask-Admin（内置）** |
| **百度收录** | 22+ 页（需优化） | **全量收录（天然支持）** |

### 2.3 不推荐方案

❌ **前后端分离（React/Vue + Flask/FastAPI）**
- SEO 不如服务端渲染
- 需要学前端框架
- 部署复杂（npm + Python）

❌ **纯 SSG（Pelican/Nikola）**
- 无后台管理
- 内容更新需重新构建

## 三、SEO 必做清单

### 3.1 基础 SEO（必须做）

| 项目 | 要求 | 说明 |
|------|------|------|
| **路由架构** | BrowserRouter（非 HashRouter） | HashRouter 搜索引擎无法抓取 # 后路由 |
| **动态 Title** | 每个页面独立 title | 产品详情页：`产品名 - 品牌名` |
| **动态 Description** | 每个页面独立 description | 产品详情页：产品描述 + 核心卖点 |
| **HTTPS** | 全站 HTTPS + HSTS | 百度收录前提 |
| **移动端适配** | 响应式设计 | 百度移动优先索引 |
| **结构化数据** | JSON-LD（Product/Organization） | 搜索引擎识别产品、价格、描述 |
| **Sitemap** | 动态生成 sitemap.xml | 包含所有产品页 |
| **Robots.txt** | 允许搜索引擎抓取 | 配置 Allow 规则 |

### 3.2 SEO 评分目标

| 指标 | 目标值 | 说明 |
|------|-------|------|
| **综合评分** | 90+/100 | 行业领先水平 |
| **内容可抓取性** | 100/100 | 所有页面可被抓取 |
| **百度收录页数** | 全量收录 | 所有产品页 + 首页 + 资讯页 |

### 3.3 SEO 优势展示（面向甲方）

**核心竞争优势：**
1. ✅ 全量产品可被搜索引擎收录（传统官网只有 3-5 页）
2. ✅ 每个产品有独立 SEO 标签（title + description）
3. ✅ 搜索引擎可以抓取所有内容（BrowserRouter 架构）
4. ✅ 移动端 + HTTPS 完美（100/100）
5. ✅ 结构化数据支持（JSON-LD）

**预期效果：**
- 百度收录：全量页面（覆盖所有产品）
- 搜索品牌名：首页第 1 位
- 搜索行业关键词：前 3-5 页

## 四、部署避坑指南

### 4.1 宝塔面板 Nginx 配置路径

❌ **错误路径：** `/etc/nginx/sites-available/`
✅ **正确路径：** `/www/server/panel/vhost/nginx/`

**示例：**
```bash
# 盛安密封官网配置
/www/server/panel/vhost/nginx/shengan-seal.conf

# 主站反向代理规则
/www/server/panel/vhost/nginx/proxy/maichewei.com/shengan.conf
```

### 4.2 Nginx location 顺序

❌ **错误：** 精确匹配放在通配匹配之后
✅ **正确：** 精确匹配必须放在最前面

**示例：**
```nginx
# ✅ 正确顺序
location = /shengan/sitemap.xml { ... }  # 精确匹配（最前）
location = /shengan/robots.txt { ... }   # 精确匹配
location /shengan/api/ { ... }           # 前缀匹配
location /shengan/ { ... }               # 通配匹配（最后）
```

### 4.3 Nginx images location 必须用 ^~ 前缀

❌ **错误：** `location /images/ { ... }`
✅ **正确：** `location ^~ /images/ { ... }`

**原因：** `^~` 确保前缀匹配优先级高于后续正则匹配（如 `location ~* \.(jpg|png|webp)$`），否则请求会被错误路由。

### 4.4 上传路径需匹配 nginx root 路径

❌ **错误：** 上传到 `/dist/images/products/`
✅ **正确：** 上传到 `/var/www/shengan-seal/images/products/`

**原因：** nginx 配置的 root 为 `/var/www/shengan-seal`，因此图片 URL `/images/products/xxx` 实际对应文件系统路径 `/var/www/shengan-seal/images/products/xxx`。

### 4.5 scp 不支持远程通配符

❌ **错误：** `scp xiaohong:/backup/* ./local/`
✅ **正确：** `scp 'xiaohong:/backup/a.conf' 'xiaohong:/backup/b.conf' ./local/`

**原因：** shell 无法展开远程 glob，必须显式列出每个文件路径。

## 五、开发避坑指南

### 5.1 Flask JSON 字段需显式 json.loads

❌ **错误：** 直接返回数据库中的 JSON 字符串
✅ **正确：** 显式解析 JSON 字符串

**示例：**
```python
# ❌ 错误
products = db.execute("SELECT * FROM products").fetchall()
return jsonify(products)  # gallery_images 是字符串

# ✅ 正确
for item in products:
    item['gallery_images'] = json.loads(item['gallery_images']) if item.get('gallery_images') else []
    item['detail_images'] = json.loads(item['detail_images']) if item.get('detail_images') else []
return jsonify(products)
```

### 5.2 sqlite3.Row 不支持 .get()

❌ **错误：** `row.get('field')`
✅ **正确：** `dict(row).get('field')`

**示例：**
```python
# ❌ 错误
row = db.execute("SELECT * FROM products WHERE id=?", (id,)).fetchone()
name = row.get('name')  # AttributeError

# ✅ 正确
row = db.execute("SELECT * FROM products WHERE id=?", (id,)).fetchone()
rd = dict(row)
name = rd.get('name')
```

### 5.3 Flask generate_token 需兼容 SQLite NOT NULL 约束

❌ **错误：** 只插入 token，不插入 user
✅ **正确：** 插入 token + user，添加 try/except 兼容旧表结构

**示例：**
```python
def generate_token(user='admin'):
    token = secrets.token_hex(32)
    try:
        db.execute("INSERT INTO admin_tokens (token, user) VALUES (?, ?)", (token, user))
    except sqlite3.OperationalError:
        # 兼容旧表结构（无 user 字段）
        db.execute("INSERT INTO admin_tokens (token) VALUES (?)", (token,))
    db.commit()
    return token
```

### 5.4 产品图片上传格式与大小限制

**支持格式：** JPG、JPEG、PNG、GIF、WebP
**大小限制：** 单张不超过 20MB

**前端校验：**
```javascript
const MAX_SIZE = 20 * 1024 * 1024; // 20MB
if (file.size > MAX_SIZE) {
  alert(`文件 ${file.name} 超过 20MB 限制`);
  return;
}
```

## 六、交付避坑指南

### 6.1 绝不主动删除任何文件

❌ **错误：** 主动删除"临时"文件、"无用"文件
✅ **正确：** 删除操作必须用户明确要求 + 用户确认

**历史教训：**
- 差点删除 pdd_images/（10M 开发素材）
- 差点删除 Astro 文件（astro.config.mjs 等）

**正确做法：**
- 创建文件可以，但删除必须用户授权
- 即使文件是"临时"的、"无用"的，也不能主动删除
- 用户说"不要动不动删除文件" = 永远不要主动删除

### 6.2 VPS 必须有完整源代码

❌ **错误：** VPS 只部署 dist 产物，不上传源代码
✅ **正确：** VPS 必须有完整源代码（含 src/、package.json、deploy.sh）

**原因：** 客户从 GitHub clone 后需要完整源代码才能独立部署。

### 6.3 本地数据保护

❌ **错误：** 删除本地开发素材（pdd_images/、audit_*.png 等）
✅ **正确：** 所有本地数据保留，直到项目完全交付

**安全边界：**
- pdd_images/（开发素材库）— 完全保留
- audit_*.png（审计截图）— 完全保留
- backend/shengan.db（本地测试库）— 完全保留
- scrape_pdd_images.py（爬虫脚本）— 完全保留

### 8.2 核心代码示例

**app.py：**
```python
from flask import Flask, render_template
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///shengan.db'
db = SQLAlchemy(app)

class Product(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100))
    description = db.Column(db.Text)
    image_url = db.Column(db.String(200))

@app.route('/')
def index():
    products = Product.query.all()
    return render_template('index.html', products=products)

@app.route('/products/<int:id>')
def product_detail(id):
    product = Product.query.get_or_404(id)
    return render_template('product_detail.html', product=product)

if __name__ == '__main__':
    app.run()
```

**templates/product_detail.html：**
```html
{% extends "base.html" %}

{% block title %}{{ product.name }} - 盛安密封{% endblock %}
{% block description %}{{ product.description[:150] }}{% endblock %}

{% block content %}
<h1>{{ product.name }}</h1>
<img src="{{ product.image_url }}" alt="{{ product.name }}">
<p>{{ product.description }}</p>
{% endblock %}
```

## 九、总结

### 9.1 核心原则

1. **SEO 优先：** 服务端渲染 > 前后端分离
2. **简单至上：** Flask + Jinja2 > React + Flask
3. **数据保护：** 绝不主动删除任何文件
4. **备份先行：** 每次改造前必须备份
5. **文档完整：** 工作日记 + 交付文档 + SEO 优势

### 9.2 推荐技术栈

**首选：Flask + Jinja2 + Tailwind CSS**
- SEO 最佳（服务端渲染）
- 开发最简单（只需 Python）
- 交付最简单（Python + nginx）
- 维护成本最低

### 9.3 避坑清单

| 坑点 | 正确做法 |
|------|---------|
| 宝塔 Nginx 路径 | `/www/server/panel/vhost/nginx/` |
| Nginx location 顺序 | 精确匹配在前，通配匹配在后 |
| Nginx images location | 用 `^~` 前缀 |
| 上传路径 | 匹配 nginx root 路径 |
| scp 远程通配符 | 显式列出每个文件 |
| Flask JSON 字段 | 显式 json.loads |
| sqlite3.Row | 先转 dict |
