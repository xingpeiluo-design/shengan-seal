- **教程里的假/错部分（必须纠正）**：
  - 安装命令 `uv tool install claude-code` → **错的**。Claude Code 是 Node/TS 应用，`uv` 是 Python 包管理器装不了它。真实安装：`npm install -g @anthropic-ai/claude-code` 或官方原生安装器 `curl -fsSL https://claude.ai/install.sh | bash`。
  - 启动命令 `claude-code` → **错的**，真实是 `claude`。
  - "先 git init + git commit init，再 /code-review" → **陷阱**：`/code-review` 只审当前未提交 diff，全提交后 diff 为空，会报"无问题"假阴性。盛安已全部提交并推送 GitHub，直接 `/code-review` 看不到任何存量代码。
- **教程里真实的部分**：
  - `/code-review [low|medium|high|xhigh|max|ultra] [--fix] [--comment] [target]` 官方语法基本属实（审当前 diff，correctness + cleanup；`--fix` 应用修复、`--comment` 发 PR 评论、`ultra` 云端多代理）。
  - `/review`、`/security-review` 官方真实存在（审 PR/分支，多代理只读审查）。
  - "三类问题靠自定义提示词驱动"是合法用法——`/code-review`、`/review` 是通用审查引擎，**没有内置的硬编码/ORM 专属检测器**；三类检测来自你给的自然语言指令。
