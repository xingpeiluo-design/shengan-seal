# 盛安密封售卖版重构与 DEPLOY 极简版补全

**关联工作日记：** 无。

## 一、需求背景

盛安密封官网定位从「内部演示项目」转为「成品网站售卖」：客户拿到源码后需在自己的服务器 + 自己的域名一键部署上线，无需修改任何代码、不依赖任何隐藏配置。

此前部署遗留两个**售卖标准下不可接受的硬伤**：

1. **API 子路径缺失**：JS `fetch()` 不认 `<base href>`，导致部署到 `/shengan/` 子路径后产品中心接口全部 404，显示"共 0 款产品"。
2. **文档只面向程序员**：DEPLOY.md 满篇 nginx/systemd/certbot 命令，90% 小白客户拿到直接劝退，售后压力集中在"我不懂命令行"。

## 二、核心原则（售卖标准·零硬编码）

- 所有静态资源：根相对路径 `/images/...`、`/api/...`
- 禁止硬编码 IP / 测试域名 / 协议头
- 客户端 JS 自动适配部署环境（根/子路径通用）
- 后端从请求头动态推断站点 URL，无需 SITE_URL 配置
- 文档双版本：极简版给老板看，技术版给程序员看

## 三、修改清单（7 个文件）

### 1. `vite.config.ts` —— 删除 base href 注入 + 简化
- 删除 `__SITE_URL__` 占位符注入逻辑
- 仅保留 `VITE_BASE_PATH` 环境变量控制（默认 `/`）
- 客户独立域名部署零配置，子路径部署仅需 1 个环境变量

### 2. `index.html` —— 移除 SEO 绝对 URL 占位符
- 删除 `__SITE_URL__` 替换逻辑
- canonical / og:url / JSON-LD 中的 URL 字段**全部移除**（避免 SEO URL 与实际部署不一致）
- 图片路径用 `/images/...` 绝对根路径

### 3. `nginx` 配置 —— 删兜底反代 + 加 IP 访问 301 跳转
- 删除 `/images/` 兜底反代（资源全部根相对，不再需要）
- 新增 `if ($host != $server_name) { return 301 https://$server_name$request_uri; }`
- 防止客户用 IP 直接打开导致资源异常 / 暴露服务器 IP

### 4. `src/api.ts` —— API base 走 `import.meta.env.BASE_URL`
- `getApiBase()` 改为读取 Vite 注入的 BASE_URL，自动包含子路径前缀
- 根部署：`/api/products`
- 子路径部署：`/shengan/api/products`
- 关键修复：JS `fetch()` 不认 `<base href>`，所以必须在 JS 层显式拼 BASE_URL

### 5. `deploy.sh` —— 默认根部署 + 子路径可选
- 默认 `VITE_BASE_PATH=/`（客户独立域名场景）
- 子路径部署：`VITE_BASE_PATH=/shengan/ bash deploy.sh`
- 顶部加 IP 访问 301 跳转的 nginx 提示注释

### 6. `DEPLOY.md` —— 从「卖源码」角度重写 + 新增极简版
- **新增「🚀 零、5 分钟快速部署（小白图文版）」章节**（用户 7/16 最新指示）
- 极简版 7 步：买服务器 → 上传源码 → 建库 → 改配置 → 一键部署 → 绑域名 → 验收
- 用 emoji + 表格 + 步骤框呈现，不依赖真实截图
- 末尾一行「👇 看不懂上面？往下翻 👇」分流程序员
- 原 220 行技术细节保留在章节二~七不变

### 7. `backend/app.py` —— SITE_URL 从请求头动态推断
- 新增 `_infer_site_url()` 函数：从 `X-Forwarded-Proto` + `X-Forwarded-Host` / `Host` 拼站点 URL
- `sitemap_xml()`、`robots_txt()` 全部走 `_infer_site_url()`
- 环境变量 `SHENGAN_SITE_URL` 仅作为可选覆盖（不设也能跑）
