## 阶段 1：竞品调研与需求确认（1-2 天）

### 1.1 明确甲方需求

**必须搞清楚的 5 个问题：**

| 问题 | 示例 | 为什么重要 |

| 网站定位 | 展示型 / 营销型 / 电商型？ | 决定功能复杂度 |
| 目标用户 | 客户群体是谁？年龄？行业？ | 决定设计风格 |

### 1.2 竞品分析（关键步骤）

**为什么要做竞品分析：**
- 知道同行网站长什么样，避免闭门造车
- 找到差异化卖点，让甲方觉得"比同行好"
- 提前规划 SEO 关键词，抢占搜索流量

**竞品分析三步法：**

1. **搜同行**：百度搜索"行业 + 产品关键词"，记录前 10 名网站
2. **拆结构**：记录每个网站的栏目设置、功能模块、设计风格
3. **找差距**：哪些做得不好（加载慢、手机不适配、没 SEO），就是我们的优势

**竞品分析表模板：**

| 竞品网站 | 栏目结构 | 设计风格 | 加载速度 | 手机适配 | SEO 评分 | 可借鉴 | 可超越 |
|----------|---------|---------|---------|---------|---------|--------|--------|
| 竞品A | 首页/产品/关于/联系 | 简约蓝色 | 快 | ✅ | 70 | 产品展示方式 | 移动端体验 |
| 竞品B | 首页/方案/案例/新闻 | 科技感 | 慢 | ❌ | 40 | 案例展示 | 全面超越 |

## 阶段 2：技术选型（0.5 天）

### 2.1 选型原则

**核心原则：匹配需求，不追新技术**

| 项目类型 | 推荐方案 | 理由 |
|----------|---------|------|
| 展示官网（卖给中小客户） | Flask + Jinja2 + Tailwind | SEO 最佳，开发最快 |
| 管理系统 / SaaS | React/Vue + API | 交互复杂，前后端分离 |
| 电商平台 | Django + PostgreSQL | 功能多，需要成熟框架 |
| AI 建站工具 | Next.js / Nuxt.js | SSR/SSG，SEO 友好 |

### 2.2 推荐方案：Flask + Jinja2 + Tailwind CSS

**为什么选这个组合（只会 Python 的开发者最优解）：**

| 维度 | Flask + Jinja2 | React SPA | Django |
|------|---------------|-----------|--------|
| **SEO** | 90+/100（服务端渲染） | 55-75/100（客户端渲染） | 85+/100（服务端渲染） |
| **开发速度** | 最快（纯 Python） | 慢（需学前端框架） | 中等（框架重） |
| **学习成本** | 低（只需 Python + HTML） | 高（JS + React + 构建工具） | 中（ORM + 模板 + 配置） |
| **部署难度** | 简单（Python + nginx） | 复杂（Node + 构建） | 中等 |
| **适合场景** | 展示官网、营销页 | 管理系统、SaaS | 大型应用、电商 |

**技术栈清单：**

```
后端：Flask 3.x + SQLite/MySQL
模板：Jinja2（Flask 自带）
样式：Tailwind CSS 4.x（CDN 引入，零构建）
图标：Font Awesome / Lucide Icons
部署：Gunicorn + Nginx
```

### 3.5 Step 6：数据库设计

**SQLite 最小表结构（展示官网够用）：**

```python
# models.py
from flask_sqlalchemy import SQLAlchemy
from datetime import datetime

db = SQLAlchemy()

class Product(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100), nullable=False)
    description = db.Column(db.Text)
    image = db.Column(db.String(200))  # 存相对路径 /static/uploads/products/xxx.jpg
    category = db.Column(db.String(50))
    sort_order = db.Column(db.Integer, default=0)
    is_active = db.Column(db.Boolean, default=True)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)

class CompanySetting(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    key = db.Column(db.String(50), unique=True, nullable=False)
    value = db.Column(db.Text)

class Admin(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(50), unique=True, nullable=False)
    password_hash = db.Column(db.String(200), nullable=False)
```

**数据库规范：**
- 图片路径只存相对路径（`/static/uploads/products/xxx.jpg`）
- 不存任何 IP、域名
- 提供初始化 SQL 脚本（含默认管理员账号 + 示例数据）

### 3.6 开发阶段避坑清单

| 坑 | 表现 | 解决方案 |
|----|------|----------|
| 硬编码 IP | 图片全裂 | 统一用根相对路径 `/static/` |
| JSON 字段不解析 | 读取报错 | `json.loads(data.field)` 显式转换 |
| sqlite3.Row 不支持 .get() | 取字典值报错 | 先转 dict：`dict(row)` |
| NOT NULL 约束 | 插入数据失败 | 建表时给默认值或允许 NULL |
| 图片上传路径不匹配 | 上传成功但显示裂图 | 上传路径必须与 nginx 静态路径一致 |
| 忘记 CSRF 保护 | 表单被恶意提交 | Flask-WTF 或手动校验 token |
| 数据库文件权限 | 写入报错 | `chmod 755 instance/` |

### 4.1 基础 SEO 清单（必须做）

| SEO 项 | 做法 | 重要度 |

| **每个页面独立 description** | Jinja2 block description，150 字以内 | ⭐⭐⭐⭐⭐ |

| **图片 alt 属性** | 所有 `<img>` 必须有 alt | ⭐⭐⭐⭐ |
| **语义化 HTML** | 用 `<header>` `<main>` `<section>` `<footer>` | ⭐⭐⭐⭐ |
| **HTTPS** | 部署 SSL 证书 | ⭐⭐⭐⭐⭐ |
| **移动端适配** | viewport meta + 响应式设计 | ⭐⭐⭐⭐⭐ |
| **URL 结构清晰** | `/products/1` 而非 `/p?id=1` | ⭐⭐⭐⭐ |
| **加载速度** | 图片压缩 WebP + Gzip + CDN | ⭐⭐⭐ |
| **sitemap.xml** | 自动生成并提交百度 | ⭐⭐⭐⭐ |
| **robots.txt** | 允许爬虫访问核心页面 | ⭐⭐⭐⭐ |

### 4.2 sitemap.xml 自动生成

```python
@app.route('/sitemap.xml')
def sitemap():
    products = Product.query.filter_by(is_active=True).all()
    xml = '<?xml version="1.0" encoding="UTF-8"?>\n'
    xml += '<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">\n'
    
    # 首页
    xml += '  <url><loc>/</loc><priority>1.0</priority></url>\n'
    # 产品列表
    xml += '  <url><loc>/products</loc><priority>0.9</priority></url>\n'
    # 每个产品详情
    for p in products:
        xml += f'  <url><loc>/products/{p.id}</loc><priority>0.8</priority></url>\n'
    
    xml += '</urlset>'
    return xml, 200, {'Content-Type': 'application/xml'}
```

### 5.2 Playwright E2E 自动化测试（推荐）

**为什么推荐 Playwright：**
- 微软出品，免费开源
- 支持 Chromium / Firefox / WebKit 三引擎
- 可模拟真实用户操作（点击、输入、上传）
- 一次编写，三引擎测试

**安装：**

```bash
pip install playwright
playwright install
```

**最小 E2E 测试脚本：**

```python
# tests/test_e2e.py
from playwright.sync_api import sync_playwright

def test_homepage():
    with sync_playwright() as p:
        browser = p.chromium.launch()
        page = browser.new_page()
        page.goto('http://localhost:5000')
        
        # 验证标题
        assert '公司名' in page.title()
        
        # 验证导航
        page.click('text=产品中心')
```

```bash
#!/bin/bash
# 企业官网一键部署脚本

# 4. 配置 Nginx

echo "=== 部署完成 ==="
```

### 6.3 Nginx 配置模板

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    
    # 静态资源
    location /static/ {
        alias /www/wwwroot/yourdomain/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # Flask 应用
    location / {
        proxy_pass http://127.0.0.1:8082;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    # 禁止 IP 直接访问（商用必备）
    if ($host != $server_name) {
        return 301 https://$server_name$request_uri;
    }
}
```
